Snow-Lang 想法及原型

有一个想法，做一款属于自己的编程语言, 让自己能按照自己的风格去写代码

特点

只要不能吃老本，那必然就有自己的亮点

极简 精致 随意

我希望它能像一片简简单单雪花一样微小，但有着精致严密的结构，却能随心所欲地捏成各种形状

极简:

极简并不是要求她的编译器或者编译产物有一个较小的大小
而且要求代码的能够清晰 优雅的达到想要的目的
追求逻辑书写和阅读上的极简, 能够简易的书写代码不是指代码量少而是结构，思虑，逻辑上的极简和阅读极高的可读性

精致:

要达到极简的目标，她就必须拥有极简的能力
稳定和创新是十分关键的

随意:

灵活的编写代码，顺利的通过编译痛苦的DEBUG，轻松发布到平台因此选择制作编译器而不是解释器

自动推断类型 无需关键词let 但为特殊情况的可读性保留

函数

大括号即函数
返回类型可自动推断
return 仅返回上一次函数 无论是匿名还是嵌套

```ts
// 完整形式
let test1:(bool,i32){str} = (b:bool,n:i32):str{

// 模式匹配 返回类型需要一致
return
  //b!?false  !?表示不等于
    b?true:{ // 当类型给bool时?true可忽略 可写作 b:{ /* ... */ }

    // 模式匹配 返回类型需要一致
    return n?0:():str{ // 空括号可忽略 返回类型str可忽略 自动推断
        return "number is 0";
    }?1:{  // 空括号可忽略
        return "number is 1";
    }?2:>"number is 2"; // 仅有返回值可变为箭头函数:>
    ?>"number not is 0 or 1 or 2"; 
    // 如果返回类型与n的类型不一致则必须添加默认case(上一行)
    // 如果返回类型与n的类型一致 再没有默认case的情况下返回n
    
    }?>"b is not true";

}

// 最简写法 完全等价 极简
